# KuaiRec 阶段六：MLU 单卡/双卡吞吐实验

## 1. 实验目标

本阶段不做完整 TopK 评测，只验证 MLU 工程能力：同一个 Two-Tower in-batch 训练任务在单卡和双卡下的吞吐差异。

## 2. 实验设置

| 项目 | 数值 |
|---|---:|
| 数据源 | `big_matrix.csv` |
| 标签 | `watch_ratio >= 0.8` |
| 训练样本 | 2,000,000 |
| Epochs | 2 |
| Loss | `inbatch` |
| 单进程 batch size | 8,192 |
| Embedding dim | 64 |
| Tower dim | 64 |
| Hidden dim | 128 |

## 3. 吞吐结果

| 配置 | 设备数 | Global batch size | 训练秒 | 处理样本 | Samples/s |
|---|---:|---:|---:|---:|---:|
| 单卡 | 1 | 8,192 | 5.5299 | 4,000,000 | 723,334.88 |
| 双卡 DDP | 2 | 16,384 | 4.4045 | 4,000,000 | 908,159.25 |

双卡相比单卡吞吐提升约 `25.6%`。

## 4. 结论

双卡 DDP 已跑通，说明当前项目具备 MLU 多卡训练适配能力。但本轮加速比不是线性 2 倍，
主要原因可能是：

- 当前 benchmark 只统计训练循环，不包含完整数据管道优化。
- 两卡 DDP 有梯度同步开销。
- 每进程 batch size 仍较大，单卡已经能跑出较高吞吐。
- CNCL 日志提示当前驱动低于 torch_mlu 镜像期望的最低版本。

该结果适合写入工程化报告：已完成单卡/双卡训练链路和吞吐对比，但后续仍可继续优化数据加载、batch size 和通信配置。
