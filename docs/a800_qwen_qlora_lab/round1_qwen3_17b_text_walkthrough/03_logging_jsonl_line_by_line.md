# `logging.jsonl` 逐行讲解

原始文件：

```text
docs/a800_qwen_qlora_lab/remote_snapshot/qwen-qlora-lab-small/outputs/qwen3_17b_text/v1-20260613-211241/logging.jsonl
```

这个文件由 `ms-swift` 在训练过程中写出。因为训练脚本设置了：

```bash
--logging_steps 1
--max_steps 20
```

所以前 20 行对应 20 个训练 step。第 21 行是训练整体汇总，第 22 行是最终元信息和完整历史记录。

## 每行字段是什么意思

训练 step 行包含这些字段：

| 字段 | 含义 | 怎么看 |
|---|---|---|
| `loss` | 当前 step 的训练损失 | 整体下降通常说明模型在拟合训练数据 |
| `grad_norm` | 梯度范数 | 太大或 NaN 说明训练可能不稳定 |
| `learning_rate` | 当前学习率 | 本轮使用 cosine 调度，逐步下降到 0 |
| `token_acc` | token 级预测准确率 | 辅助观察训练是否学到模式 |
| `epoch` | 数据集遍历进度 | 不是整数也正常，因为只跑了 20 步 |
| `global_step/max_steps` | 当前步数/总步数 | 本轮是 `1/20` 到 `20/20` |
| `elapsed_time` | 已运行时间 | 用来估算训练耗时 |
| `remaining_time` | 预计剩余时间 | 只是估算值 |
| `memory(GiB)` | 框架记录显存 | 约 `3.82 GiB`，和外部 nvidia-smi 口径不同 |
| `train_speed(s/it)` | 每步耗时 | 越低越快 |

## 从这份日志能得出什么结论

可以得出的结论：

1. 训练跑满了 20 步。
2. loss 从 `7.39` 降到 `2.64` 左右，训练集拟合明显。
3. 学习率按 cosine 调度从 `1e-4` 降到 `0`。
4. 每步耗时约 3.9-4.7 秒。
5. 框架记录显存约 `3.82 GiB`。
6. 最终 checkpoint 是 `checkpoint-20`。

不能得出的结论：

1. 不能证明模型泛化能力好。
2. 不能证明微调后一定优于 base 模型。
3. 不能证明这批数据质量高。
4. 不能证明可以生产部署。

原因是：本轮训练数据很小，而且没有独立验证集。

## 第 22 行重点解释

第 22 行很长，里面最重要的是：

```text
PeftModelForCausalLM: 1729.2913M Params (8.7163M Trainable [0.5040%])
```

意思是：

| 数值 | 含义 |
|---|---|
| `1729.2913M Params` | 模型总参数约 17.29 亿 |
| `8.7163M Trainable` | 本轮真正训练的参数约 871 万 |
| `0.5040%` | 可训练参数只占总参数约 0.5% |

这正是 LoRA/QLoRA 的核心价值：不用更新全部 17 亿参数，只训练很小的 adapter 参数。
